Movella DOT PC SDK API 2023.6.0
Code Examples
The SDK contains a set of example applications that show how to interface with the Movella DOT Device API. Next to this, the examples also show how you could import the SDK and use them in the available programming languages. More information can be found in the Example folder and the examples itself.

These examples can be found in the Examples directory of the Movella DOT PC SDK:

<Movella DOT PC SDK installation directory>/Examples

Movella DOT Device API Examples

The Movella DOT Device API (MDDA) Examples are split into the following:
  • Read Data from Devices and set some values
  • Write Data to Devices
  • Do a firmware update for a Device from the Update Server
  • Perform a magnetic field mapping of a Device
  • Make an onboard recording on a Device
  • Export a recording from a Device
  • Sync measurement using 2 or more devices
They are all command line applications with limited user interaction, available in several programming languages: C++, C#, Java and Python. The structure of these examples is more or less the same in all available programming languages.

Example Read shows how to do following things:
  • Scan for Movella DOT Devices
  • Use Callbacks to receive found Devices
  • "Open the port" Connect to Movella DOT Device
  • Read TagName and Firmware Version from a Connected Movella DOT Device
  • Receive battery notifications on change in Charging state and Level
  • Print received error notifications onError
  • Read the incoming data from Movella DOT Device
  • Do a Heading Reset
  • Clean up
Example write shows how to do following things:
  • Scan for Movella DOT Devices
  • Use Callbacks to receive found Devices
  • "Open the port" Connect to Movella DOT Device
  • Read TagName from a Connected Movella DOT Device
  • Write a new tagname to the device using setTagName
  • Print received error notifications onError
  • Clean up
Example Firmware Update shows how to do following things:
  • Scan for Movella DOT Devices
  • Use Callbacks to receive found Devices
  • "Open the port" Connect to Movella DOT Device
  • Read TagName from a Connected Movella DOT Device
  • Start Firmware Update from the Server for the current device
  • Print received progress updates through callback and wait for the Done callback
  • Clean up
Example Magnetic Field Mapper shows how to do following things:
  • Scan for Movella DOT Devices
  • Use Callbacks to receive found Devices
  • "Open the port" Connect to Movella DOT Device
  • Start Magnetic field mapping of Movella DOT Device
  • Print received progress notifications through callback and wait for 100 percent
  • Stop the Magnetic Field Mapping. This stores the results to the Movella DOT Device when valid
  • Clean up
Example Onboard Recording shows how to do following things:
  • Scan for Movella DOT Devices
  • Use Callbacks to receive found Devices
  • "Open the port" Connect to Movella DOT Device
  • Set the output rate the onboard recording should use
  • Start the (timed) onboard recording on the Movella DOT Device
  • Print requested timing information while the recording is running
  • Clean up
Example Data Export shows how to do following things:
  • Scan for Movella DOT Devices via USB
  • Open the USB COM port to connect to Movella DOT Device
  • Set the export data to export from the Device
  • Start the export of a file and wait for the Export Done callback
  • Clean up
Example Synchronization shows how to do following things:
  • Scan for Movella DOT Devices
  • Use Callbacks to receive found Devices
  • "Open the port" Connect to Movella DOT Device
  • Match the output rate and filter profile of devices to be synchronized
  • Start sync with the specified root node
  • Start measurement and print some data
  • check the timestamp column to check first root node timestamp also present in other data streams
  • Clean up
Note
Besides updating from the the server it is also possible to checkForUpdates and start an update from a file. See XsDotDevice functions.
